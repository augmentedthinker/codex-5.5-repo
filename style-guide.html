House Style

This is the reference page for how the Codex 5.5 site should look and behave as it grows. New pages should reuse the shared stylesheet and navigation patterns unless Christopher asks for a deliberate exception.

Architecture

The shared site styling lives in assets/css/site.css. Standard pages should link that file and avoid duplicating global CSS. Page-specific CSS is still allowed for special artifacts, animations, or one-off experiments.

  • .site-header and .site-nav define the universal navigation bar.
  • .shell, .shell.narrow, and .shell.readable define page width and spacing.
  • .hero, .eyebrow, .description, and .card-link define the main site language.

Navigation

The top navigation should remain internal to the site: Home, README, Agents, Style, Artifacts, and Memories. External links should usually live inside content pages rather than the universal nav, so the user does not unexpectedly leave the site.

Page Pattern

Most pages should start with a hero card containing an eyebrow label, a large title, and a justified paragraph describing the page. Below that, use card links for archive entries or content cards for long-form material.

Archive pages should be newest-first. Artifact and memory entries should include a date/time/model signature when the content is generated by a model.

When Adding New Features

  • Add reusable styles to assets/css/site.css only when at least two pages benefit from them.
  • Keep visual artifacts free to be expressive, but preserve the universal nav unless the artifact intentionally needs full immersion.
  • Update this page whenever we introduce a reusable layout, component, or convention.
  • Update README.md when architecture changes affect future maintenance.
