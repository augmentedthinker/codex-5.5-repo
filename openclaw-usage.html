How Ash calculates this

Plain English: Ash asks OpenClaw for the private Codex usage snapshot. That snapshot reports how much of each window is used, when each window resets, and how long each window is. Ash publishes only the safe summary here: percent used, percent of time elapsed, and start/end times.

Important: this page does not publish OAuth tokens, account IDs, raw provider responses, or hidden quota details. It is a sanitized visual copy of the useful parts.

  • Usage percent: comes directly from the Codex/OpenAI OAuth usage snapshot as usedPercent. Example: 80% means 80% of that quota window has been used.
  • End / reset time: comes directly from the snapshot as resetAt. That is when the provider says the window resets.
  • Start time: is calculated as resetAt − window duration. For the short window, that duration is 5 hours. For the weekly window, that duration is 7 days.
  • Time elapsed ring: is calculated as (now − start) ÷ window duration. That lets us compare burn rate against how far through the window we are.

Technical note: on the OpenClaw host, Ash can use OpenClaw’s internal provider-usage loader for openai-codex. The loader returns sanitized fields like provider name, plan label, usage windows, usedPercent, and resetAt. Ash then converts reset timestamps into America/New_York display times and updates this page.

How to use this page

  • Source: Ash uses OpenClaw’s private provider-usage loader for openai-codex, which reads the OAuth-backed Codex usage snapshot from the ChatGPT usage backend. Raw tokens, account IDs, and provider responses are not published here.
  • Window math: the source provides used percentage, reset time, and window duration. The displayed start time is calculated as reset time − window duration; the displayed end time is the provider reset time. The secondary green ring shows elapsed time in the current window, so usage can be compared against pace.
  • Update flow: ask Ash for a “usage dashboard update.” Ash should check current usage, update the gauges, start/end labels, and last-updated pill, commit the change, push it to GitHub, and verify the live page.
